# 第五步：重写 SKILL.md

## 你的任务

编写轻量级、聚焦的 SKILL.md 作为总控文件。

## SKILL.md 目标

### 行数控制
- **目标行数**：50-80 行
- **精简原则**：只保留核心内容，详细内容移到其他文件

### 必须包含的内容

#### 1. Frontmatter（YAML 元数据）
```yaml
---
name: skill-name
description: [清晰的描述，说明 skill 做什么，何时使用]
license: Complete terms in LICENSE.txt (可选)
---
```

#### 2. 概述（3-5 段落）
- Skill 简要说明（1-2 段落）
- 核心价值/用途（1-2 段落）
- 用户触发条件描述（1-2 段落）

#### 3. 工作流索引（表格）
| 步骤 | 说明 | 详细指引 |
|:---:|:---:|:---|
| 1️⃣ | 步骤名称 | [workflow/step1-xxx.md](workflow/step1-xxx.md) |
| 2️⃣ | 步骤名称 | [workflow/step2-xxx.md](workflow/step2-xxx.md) |
| ... | ... | ... |

#### 4. ⚠️ 执行前必读
列出核心规则文件（必须读取）：
- [rules/禁止清单.md](rules/禁止清单.md) - 绝对禁止事项
- [rules/写作规范.md](rules/写作规范.md) - 语言风格要求

#### 5. 📚 参考资料（按需查阅）
列出参考文件（按需加载）：
- [references/API文档.md](references/API文档.md) - 详细 API 说明
- [references/示例.md](references/示例.md) - 使用示例

#### 6. 🚀 快速开始
提供 2-3 个常见使用场景：
```
场景一：[描述]
执行：步骤1 → 步骤2 → ...
```

#### 7. 📋 文件位置（可选）
如果有固定的输入/输出路径，说明：
```
输入：xxx
输出：xxx
```

## 不应包含的内容

### ❌ 避免嵌入
- 长篇代码示例（移到 scripts/ 或 templates/）
- 详细的 API 文档（移到 references/）
- 完整的配置说明（移到 rules/ 配置说明.md）
- 详细的步骤说明（已在 workflow/ 中）

### ❌ 避免冗余
- 重复的信息
- 过长的说明
- 不必要的细节

## 写作风格

### 语气
- 直接、简洁、命令式
- 使用"你的任务"、"执行步骤"、"完成标志"等明确短语

### 格式
- 使用表格展示工作流
- 使用列表展示参考资料
- 使用加粗强调关键信息

### 链接
- 使用相对路径链接（如 `[workflow/step1-xxx.md](workflow/step1-xxx.md)`）
- 确保所有链接正确

- [ ] 包含所有必需部分（frontmatter、概述、工作流索引、参考资料）
- [ ] 不包含详细内容（已在其他文件中）
- [ ] 所有链接路径正确
