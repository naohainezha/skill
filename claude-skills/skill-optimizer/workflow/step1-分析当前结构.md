# 第一步：分析当前结构

## 你的任务

分析待优化的 skill 的当前结构，识别问题和优化空间。

## 分析清单

### 1. 读取 SKILL.md
- 统计行数
- 检查是否有 frontmatter
- 识别内容类型（工作流/知识库/混合型）

### 2. 检查目录结构
- 列出所有文件和目录
- 识别现有模块（workflow/rules/references/templates/scripts/assets）
- 统计文件数量

### 3. 识别问题类型
- **上下文过载**：SKILL.md 行数 > 100
- **缺少模块化**：内容混在单个文件中
- **工作流不清晰**：步骤描述分散，无独立文件
- **规则分散**：禁止事项、规则混在文本中
- **代码嵌入**：代码示例直接写在文档中

### 4. 确定 skill 类型
- **工作流型**：多步骤任务（如 xiaohongshu-ai-workflow）
- **工具型**：工具调用（如 xhs-downloader）
- **知识库型**：静态知识（如 xhs-copywriter）
- **混合型**：多种类型混合

## 完成标志

完成分析报告，包含：
- 当前结构概览（目录树）
- SKILL.md 行数统计
- 识别的主要问题（列出 3-5 个）
- 推荐的优化方向

进入下一步设计新架构。
