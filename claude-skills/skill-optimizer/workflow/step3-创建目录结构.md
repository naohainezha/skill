# 第三步：创建目录结构

## 你的任务

根据设计的架构，创建 skill 的目录结构。

## 创建步骤

### 1. 创建基础目录
```bash
mkdir -p "skill-path/workflow"
mkdir -p "skill-path/rules"
mkdir -p "skill-path/references"
mkdir -p "skill-path/templates"
mkdir -p "skill-path/scripts"
mkdir -p "skill-path/assets"
```

### 2. 验证目录创建
确认所有目录已成功创建。

## 目录说明

### workflow/ - 工作流步骤
- 每个步骤一个独立 Markdown 文件
- 文件命名：`stepN-步骤名.md`（N 从 1 开始）
- 内容结构：
  ```markdown
  # 第N步：步骤名
  
  ## 你的任务
  [任务描述]
  
  ## 执行步骤
  1. [具体步骤]
  2. [具体步骤]
  ...
  
  ## 完成标志
  [完成条件]
  
  ## 参考资源
  - [引用相关文件]
  ```

### rules/ - 规则和要求
- 规则文件独立，清晰分类
- 推荐文件：
  - `禁止清单.md`：绝对禁止的词汇和行为
  - `写作规范.md`：语言风格和内容要求
  - `字数要求.md`：严格的字数限制
  - `配置说明.md`：配置参数详解

### references/ - 详细参考资料
- 详细文档、API 参考、示例
- 推荐文件：
  - `API参考.md`：API 方法详解
  - `项目结构.md`：项目/工具的结构说明
  - `示例.md`：详细使用示例
  - `常见问题.md`：问题和解决方案

### templates/ - 输出模板
- 输出格式、脚本模板
- 推荐文件：
  - `输出格式.md`：标准输出结构
  - `配置模板.md`：配置文件示例

### scripts/ - 工具脚本（可选）
- 可执行的 Python/Bash 脚本
- 用途：自动化任务、模板生成
- 特点：可直接执行，无需加载到上下文

### assets/ - 静态资源（可选）
- 图片、图标、字体、配置文件
- 用途：输出中使用，不加载到上下文

## 完成标志

所有目录创建完成，进入下一步拆分内容。

## 注意事项

⚠️ **目录命名规范**：
- 使用中文或英文，保持一致性
- 避免特殊字符和空格
- 使用连字符 `-` 连接多词

⚠️ **权限检查**：
- 确保对 skill-path 有写权限
- 如遇权限问题，使用管理员权限
