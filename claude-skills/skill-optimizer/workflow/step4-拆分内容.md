# 第四步：拆分内容

## 你的任务

将原始 SKILL.md 的内容拆分到对应的新文件中。

## 拆分策略

### 1. 提取 workflow 步骤
从 SKILL.md 中识别工作流步骤，为每个步骤创建独立文件。

**识别特征**：
- "步骤一"、"步骤 1"、"1️⃣"
- "Step 1"、"First Step"
- 编号的步骤描述

**拆分到**：`workflow/stepN-步骤名.md`

**内容结构**：
```markdown
# 第N步：步骤名

## 你的任务
[任务描述]

## 执行步骤
[具体步骤]

## 完成标志
[完成条件]

## 参考资源
- [引用相关文件]
```

### 2. 提取 rules
从 SKILL.md 中识别规则性内容。

**识别特征**：
- "禁止"、"不要"、"❌"
- "必须"、"应该"、"✅"
- "要求"、"规范"、"规则"

**拆分到**：`rules/对应名称.md`

**内容分类**：
- 禁止清单：禁止的词汇、行为、格式
- 写作规范：语言风格、语气、内容要求
- 字数要求：严格的字数限制
- 配置说明：参数详解、使用方法

### 3. 提取 references
从 SKILL.md 中识别详细参考内容。

**识别特征**：
- 长篇示例（>50 行）
- API 文档（方法、参数、返回值）
- 详细说明（步骤详解、配置说明）

**拆分到**：`references/对应名称.md`

**保留在 SKILL.md 中的内容**：
- 前言/概述（3-5 段落）
- 快速开始/使用指南
- 核心流程总结

### 4. 提取 templates
从 SKILL.md 中识别模板性内容。

**识别特征**：
- 输出格式示例
- 代码模板（可复用的代码片段）
- JSON/TXT 格式规范

**拆分到**：`templates/对应名称.md`

**模板结构**：
```markdown
# [模板名称]

## 输出位置
[文件路径]

## 输出格式
[格式说明]

## 格式示例
```
[示例内容]
```
```

### 5. 提取 scripts
从 SKILL.md 中识别代码脚本。

**识别特征**：
- 可执行的 Python/Bash 代码块
- 工具类代码
- 模板脚本

**拆分到**：`scripts/对应名称.py` 或 `.sh`

**脚本结构**：
```python
"""
脚本说明
"""
import ...

def main():
    # 实现逻辑
    pass

if __name__ == "__main__":
    main()
```

## 拆分原则

### 优先级
1. **workflow** > **rules** > **references** > **templates** > **scripts**
2. **高频使用**的内容优先拆分
3. **低频使用**的内容可保留在 SKILL.md

### 保留在 SKILL.md 中的内容
- Frontmatter（YAML 元数据）
- 概述/说明（3-5 段落）
- 工作流索引（表格）
- 快速开始/使用指南
- 核心要点总结

### 移到新文件的内容
- 详细的步骤说明
- 完整的代码示例
- API 文档
- 配置说明
- 长篇示例

## 完成标志

所有内容拆分完成，每个内容都分配到对应的新文件中。

## 注意事项

⚠️ **避免重复**：
- 检查新文件中是否有重复内容
- 确保 SKILL.md 和新文件内容不重叠

⚠️ **保持引用关系**：
- 在新文件中引用 SKILL.md（如需要）
- 在 SKILL.md 中引用新文件（按需读取）
